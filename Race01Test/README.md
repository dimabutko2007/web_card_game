# 🃏 Dark Realms - Card Game

Цей проект — це карткова онлайн-гра на тему Minecraft, побудована на Node.js, Express, Socket.io та MySQL.

## 🚀 Швидкий старт для розробників

Якщо ви щойно склонували цей репозиторій, виконайте наступні кроки, щоб запустити гру локально.

### 1. Встановлення залежностей
Відкрийте термінал у кореневій папці проекту та виконайте:
```bash
npm install
```

### 2. Налаштування бази даних (MySQL)
Переконайтеся, що у вас встановлено MySQL Server.

Виконайте в терміналі powershell: 
```powershell
Get-Content db.sql | mysql -u root -p
```

або:

1. Увійдіть у вашу консоль MySQL:
   ```bash
   mysql -u root -p
   ```
2. Виконайте імпорт схеми та даних з файлу `db.sql`:
   ```bash
   source db.sql
   ```
   *(Або скопіюйте вміст `db.sql` та виконайте його у вашому GUI-клієнті, наприклад MySQL Workbench або phpMyAdmin).*

### 3. Конфігурація підключення
Наразі параметри підключення знаходяться у файлі `src/config/config.js`. Відкрийте його та вкажіть ваш пароль до MySQL:

```javascript
// src/config/config.js
DB: {
    HOST: 'localhost',
    USER: 'root',
    PASS: 'ВАШ_ПАРОЛЬ_ТУТ', // Змініть на ваш локальний пароль
    NAME: 'card_game',
    PORT: 3306
}
```

### 4. Запуск проекту
Для розробки (з автоматичним перезавантаженням через `nodemon`):
```bash
npm run dev
```

Для звичайного запуску:
```bash
npm start
```

Після цього відкрийте браузер за адресою: `http://localhost:3000`

> **💡 Порада:** Щоб зіграти сам із собою (протестувати матч), вам потрібно відкрити другу вкладку в режимі **Інкогніто** або скористайтеся **іншим браузером**, щоб створити дві окремі сесії користувачів.

## 🛠 Технологічний стек
- **Backend:** Node.js, Express
- **Real-time:** Socket.io
- **Database:** MySQL
- **Frontend:** EJS, Vanilla CSS, JS
- **Auth:** Express-session, BCrypt

## 📂 Структура проекту

Проект має чітку архітектуру, розділену на серверну (backend) та клієнтську (frontend) частини.

### Директорія `src/`

- **`index.js`** — Головна точка входу в програму на сервері. Налаштовує Express сервер, підключає Middleware, шаблонізатор EJS, роути та ініціалізує Socket.io для взаємодії в реальному часі.

- **`config/`** — Налаштування проекту.
  - `config.js` — Зберігає конфігураційні константи, зокрема параметри підключення до бази даних.
  - `db.js` — Відповідає за встановлення з'єднання з базою даних MySQL за допомогою пулу з'єднань (mysql2).

- **`controllers/`** — Містить бізнес-логіку обробки HTTP-запитів.
  - `authController.js` — Логіка реєстрації, авторизації, виходу та скидання пароля користувачів. Отримує дані від маршрутів, взаємодіє з моделями та повертає відповідь клієнту.

- **`models/`** — Моделі для взаємодії з базою даних.

- **`routes/`** — Визначення HTTP-маршрутів (ендпоінтів).
  - `auth.js` — Маршрути для аутентифікації користувачів. Викликають відповідні методи з `authController.js`.
  - `game.js` — Маршрути для ігрових сторінок (лобі, гра, профіль), а також налаштування multer для завантаження аватарів.

- **`sockets/`** — Логіка WebSocket для реального часу.
  - `gameSocket.js` — Центральний файл обробки ігрового процесу. Керує подіями підключення, створенням кімнат, пошуком супротивника (matchmaking), ходами в матчі та синхронізацією стану гри між клієнтами.

- **`views/`** — EJS-шаблони для відображення HTML на сервері.

- **`public/`** — Статичні файли клієнтської частини, які безпосередньо доступні браузеру.
  - **`css/`** — Файли каскадних таблиць стилів (`main.css` для загальних стилів, `game.css` для специфічних стилів гри).
  - **`js/`** — Клієнтські JavaScript-сценарії.
    - `game.js` — Логіка, що виконується в браузері під час гри. Керує інтерфейсом, анімаціями та зв'язком із сервером через Socket.io.
  - **`assets/`** — Статичні графічні ресурси (наприклад, зображення карток у `cards/`, дефолтні аватари).
  - **`uploads/`** — Директорія (створюється автоматично) для збереження файлів, завантажених користувачами (наприклад, `avatars/`).

### Зв'язки та потік даних (Архітектура)
1. **HTTP Запити (Навігація):** Коли користувач відкриває певну URL-адресу (наприклад, `/profile`), запит потрапляє до `src/index.js`, який направляє його у відповідний роут у `routes/game.js`. Роут взаємодіє з `models/User.js` для отримання даних користувача з БД (`config/db.js`), а потім рендерить шаблон `views/profile.ejs` з цими даними.
2. **Аутентифікація:** Дані з форм входу/реєстрації відправляються POST-запитами на роути `routes/auth.js`, які передають виконання у `controllers/authController.js`. Контролер перевіряє дані, хешує паролі за допомогою BCrypt, взаємодіє з `models/User.js` і встановлює сесію користувача.
3. **Ігровий процес (Real-time):** Коли гравець опиняється на сторінці гри (`/game`), браузер завантажує `public/js/game.js`. Цей клієнтський скрипт ініціює WebSocket з'єднання. На сервері його приймає `sockets/gameSocket.js`. Кожна дія гравця (вибір карти, закінчення ходу) відправляється як подія Socket.io з `public/js/game.js` до `sockets/gameSocket.js`. Сервер обробляє логіку, перевіряє правила та розсилає оновлений стан гри назад обом клієнтам, які, в свою чергу, оновлюють UI на екрані.
